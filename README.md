# oil_analysis — Дашборд анализа масла (горная техника)

Рабочий репозиторий дашборда спектрального анализа масла + отказов техники.
Перенесено из `victorekuznetsov/nte200` (ветка `claude/qsk50-predictive-dashboard-XPVP6`).

## Что тут на самом деле

Фактически развёрнутый отчёт — **не** Streamlit-приложение из `OIL_DASHBOARD_SPEC.md`
(тот документ — черновик раннего дизайна и частично устарел), а один
самодостаточный HTML-файл `dashboard_report.html`: вся база проб сжата
(gzip+base64) и встроена прямо в файл, графики отрисованы кастомным canvas-движком
без внешних библиотек. Открывается двойным кликом в любом браузере, не требует
сервера. Текущая версия лежит внутри `для_другого_ПК.zip` (готова к рассылке).

Отчёт совмещает два источника:
- **Анализ масла** — `Свод_анализов_масла*.xlsm` (лист «Анализы»)
- **Отказы/ремонты** — `Сводный анализ ПК.xlsx` / `Сводный анализ NTE.xlsx`
  (лист «Сводные данные»), накладываются как маркеры на графики износа

## Содержимое

| Файл | Описание |
|------|----------|
| `dashboard_report.html` | *(не в git, см. ниже)* — актуальный отчёт, результат `automation/refresh_report.py` |
| `для_другого_ПК.zip` | Готовый отчёт для рассылки / бутстрап шаблона для автоматизации |
| `automation/` | Скрипты для автоматического обновления отчёта из свежих Excel-файлов — см. `automation/README.md` |
| `OIL_DASHBOARD_SPEC.md` | Черновик исходной спецификации дашборда (столбцы, пороги, диагностические паттерны) — частично устарел, но полезен как справочник по столбцам/порогам |
| `Свод_анализов_масла.xlsm` | База проб до 2025-12-31 (лист `Анализы`, 60 столбцов, header в строке 2) |
| `Свод_анализов_масла 2026.xlsm` | Инкремент за 2026 год (header в строке 1, без столбцов «Примечание»/«Доливка») |
| `Сводный анализ ПК.xlsx` / `Сводный анализ NTE.xlsx` | Журнал отказов/ремонтов по всему парку / по NTE, с черновой автоклассификацией Узел/Система/Воздействие (см. лист «Методика и легенда» в NTE-файле) |
| `Доливки Горная Евразия.xlsx` | Доливки масла (сырьё для будущего оверлея) |

## Обновление отчёта (автоматизация)

```bash
pip install -r requirements.txt
python3 automation/refresh_report.py
```

Одна команда: сама находит все `Свод_анализов_масла*.xlsm` и файл отказов,
объединяет их, пересчитывает все метрики (WearIdx/ContamIdx/FP/z-оценки/скользящие
средние/пороги p90-p95) и пересобирает `dashboard_report.html`. При первом запуске
на чистом клоне сама распакует шаблон из `для_другого_ПК.zip`. Подробности и
внутреннее устройство — в `automation/README.md`.

## Что дальше

- Открытые вопросы по методике/порогам — см. §7 в `OIL_DASHBOARD_SPEC.md`.
- Автоклассификация Узел/Система/Воздействие в файлах отказов помечена как
  «черновая» и не проверена (см. `Сводный анализ NTE.xlsx`, лист «Методика и
  легенда»); в `Сводный анализ ПК.xlsx` не классифицировано ~21% записей —
  кандидат на дальнейшую автоматизацию (NLP по столбцу «Описание неисправности»).
